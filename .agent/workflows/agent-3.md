# Агент 3 — Автопарк + Ремонты

Ты работаешь в монорепо `d:\Ai\TMS\`. Фундамент уже создан (Агент 0):

## Задачи

### Backend (`apps/api/src/modules/fleet/` и `apps/api/src/modules/repairs/`)

1. **Карточка ТС**:
   - API: `GET/POST/PUT /api/fleet/vehicles`, `GET /api/fleet/vehicles/:id`
   - Полный CRUD с историей
   - Сроки документов: техосмотр, ОСАГО, ТО, тахограф, пропуска
   - Светофор: >30 дней=зелёный, 7–30=жёлтый, <7=красный, просрочен=блокировка
   - Автоуведомления при приближении к сроку
   - Расход топлива: факт vs ГСМ-норма (учёт сезона, загрузки, типа ТС)
   - Пробег по телематике (placeholder для интеграции)

2. **Карточка водителя**:
   - API: `GET/POST/PUT /api/fleet/drivers`, `GET /api/fleet/drivers/:id`
   - Допуски, РТО, медсправки, привязка к ТС, штрафы
   - Валидация: формат ВУ, срок

3. **Справочники** (`apps/api/src/modules/fleet/references/`):
   - Контрагенты: API `GET/POST/PUT /api/fleet/contractors`
     - Дубликаты: предупреждение при существующем ИНН
     - Валидация ИНН (контрольная сумма)
     - Интеграция DaData (placeholder — функция `lookupByInn()`)
   - Адреса: нормализация, геокодирование (placeholder)
   - Зоны ограничений: МКАД, ТТК, городские
   - Типы грузов

4. **Ремонты**:
   - API: `GET/POST/PUT /api/repairs`
   - Стейт-машина: Создана → Ждёт з/ч → В работе → Готово
   - Источники: автоматически (из недопуска), водитель, механик, по графику ТО
   - Фиксация: вид работ, запчасти, стоимость, пробег
   - При завершении: ТС → «Доступно»
   - Плановое ТО: автозаявки по графику (км или дата)
   - События: `repair.created`, `repair.status_changed`, `repair.completed`

5. **Пропуска**:
   - API: `GET/POST/PUT /api/fleet/permits`
   - Привязка к ТС, зоны, сроки
   - Контроль сроков, уведомления о продлении

6. **Штрафы ГИБДД**:
   - API: `GET/POST/PUT /api/fleet/fines`
   - Стейт-машина: Новый → Подтверждён → Оплачен / Обжалован
   - Привязка к ТС (госномер) и водителю (по дате/времени)
   - Аналитика по ТС, водителям, типам

7. **Зарегистрируй роуты** в `apps/api/src/server.ts`

### Frontend

8. **Дашборд автопарка** (`apps/web/src/app/fleet/page.tsx`):
   - Таблица ТС: статус, госномер, марка, сроки (светофор)
   - Карточка ТС: полная информация, история ремонтов, документы
   - Таблица водителей
   - Управление пропусками и штрафами

9. **Дашборд ремонтной службы** (`apps/web/src/app/repair/page.tsx`):
   - Канбан: Создана → Ждёт з/ч → В работе → Готово
   - Описание, фото, приоритет, исполнитель
   - Фиксация работ и запчастей
   - Итоги затрат по ТС и периодам

## Правила
- **Файлы**: ТОЛЬКО `apps/api/src/modules/fleet/`, `apps/api/src/modules/repairs/`, `apps/web/src/app/fleet/`, `apps/web/src/app/repair/`
- **Не трогай**: другие модули, shared, auth, layout
- **Валидация**: ИНН (контрольная сумма), госномер (формат А000АА00), VIN (17 символов)
- **Архивирование**: удалённые записи → статус «Архив», не физическое удаление
