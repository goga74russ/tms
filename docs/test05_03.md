## Рекомендации по исправлению

| # | Проблема | Файл(ы) | Как чинить |
|---|---|---|---|
| 1 | **Создание заявки не отправляет POST** | `apps/web/src/app/logist/components/CreateOrderModal.tsx` | Проверить `onCreate` callback — вероятно, вызывается mock-функция или `api.post('/orders', data)` не подключен. Нужно убедиться, что при нажатии «Создать» отправляется `api.post('/orders', payload)` и обновляется список. |
| 2 | **Кнопка «Фильтры» не работает** | `apps/web/src/app/logist/page.tsx` | Проверить обработчик onClick — вероятно, `setShowFilters(true)` не переключает видимость панели фильтров, или компонент фильтров не реализован. |
| 3 | **Финансы: 500 ошибка API** | `apps/api/src/modules/finance/routes.ts`, `finance/service.ts` | Проверить серверные логи (`docker logs tms-api-1`). Скорее всего, ошибка в SQL-запросе к таблице invoices (нет данных / неверная схема / migration не прошла). |
| 4 | **Кнопка «Добавить водителя» не работает** | `apps/web/src/app/drivers/page.tsx` | Кнопка — просто `<button>` без onClick. Нужно создать `CreateDriverModal` с формой (ФИО, номер ВУ, категории, сроки) и подключить `api.post('/fleet/drivers', data)`. |
| 5 | **Кнопка «Добавить контрагента» не работает** | `apps/web/src/app/contractors/page.tsx` | Аналогично #4 — нужен `CreateContractorModal` с полями (Наименование, ИНН, КПП, адрес, телефон, email) и `api.post('/fleet/contractors', data)`. |
| 6 | **Кнопка «Новая заявка» на ремонтах не работает** | `apps/web/src/app/repair/page.tsx` или `RepairKanban.tsx` | Проверить `onClick` кнопки — вероятно, модал не реализован. Нужен `CreateRepairModal` с выбором ТС и описанием поломки + `api.post('/repairs', data)`. |
| 7 | **«Новый рейс» не грузит водителей** | `apps/web/src/app/logist/components/CreateTripModal.tsx` | API `/trips/available-drivers` возвращает пустой массив, т.к. нет водителей в БД. Не баг — нужно сначала создать водителей. |
| 8 | **Медосмотр: 403 для admin** | `apps/api/src/modules/inspections/routes.ts` | RBAC ожидает роль `medic` для доступа к очереди медосмотра. Для admin нужно добавить `admin` в allowedRoles OR зайти под medic@tms.local. |
