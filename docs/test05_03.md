## Исправленные баги

### 1. Создание заявки (POST /orders) — ранее 400 Validation Error
**Причина:** Модал отправлял `contractorName` (строка), а API ожидает `contractorId` (UUID).  
**Исправление:** Заменено текстовое поле на dropdown-селектор контрагентов из БД. Параметр `createdBy` инжектится из JWT на бэкенде.  
**Файл:** `apps/web/src/app/logist/components/CreateOrderModal.tsx`

### 2. Финансы 500 ошибка (GET /finance/invoices)
**Причина:** `with: { contractor: true }` — связь не определена в Drizzle ORM.  
**Исправление:** Убрана несуществующая связь из запроса.  
**Файл:** `apps/api/src/modules/finance/routes.ts`

### 3. Создание ремонта (POST /repairs) — 400 Validation Error
**Причина:** `priority: 'normal'` (не в enum), отсутствует обязательное поле `source`.  
**Исправление:** `normal` → `medium`, добавлено `source: 'mechanic'`.  
**Файл:** `apps/web/src/app/repair/page.tsx`

### 4-6. Нерабочие кнопки «Добавить»
Кнопки на страницах `/drivers`, `/contractors`, `/repair` не имели `onClick`.  
Добавлены полноценные модалы создания с API-вызовами:
- `CreateDriverModal` → POST /fleet/drivers
- `CreateContractorModal` → POST /fleet/contractors
- `CreateRepairModal` → POST /repairs

### 7. Медосмотр 403 для admin
**Причина:** RBAC ограничивал admin от `MedInspectionDetails` и `KPI`.  
**Исправление:** `can('manage', 'all')` для admin (TODO: ограничить перед production).  
**Файл:** `apps/api/src/auth/rbac.ts`
