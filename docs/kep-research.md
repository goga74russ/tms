# КЭП (Квалифицированная электронная подпись) — Исследование

> Дата: 04.03.2026 | Агент 8 | Sprint 5

## Что такое КЭП

КЭП — квалифицированная электронная подпись, юридически эквивалентная собственноручной. Требуется для подписания ЭТрН и других электронных перевозочных документов.

## Варианты для TMS

### Вариант 1: CryptoPro CSP + Node.js (серверная подпись)

**CryptoPro CSP 5.0** — криптопровайдер для работы с ГОСТ-алгоритмами (обязателен для КЭП в РФ).

| Характеристика | Значение |
|---------------|----------|
| Версия | 5.0.12000+ (обязательно с 01.10.2024) |
| Алгоритмы | ГОСТ 34.12-2018, ГОСТ 34.13-2018, ГОСТ Р 34.10-2012 |
| Лицензия сервер | ~18 000 ₽ (бессрочная) |
| Лицензия клиент | ~2 700 ₽ (бессрочная) |

**Node.js библиотеки:**

| Пакет | Назначение | Статус |
|-------|-----------|--------|
| `n-cryptopro` | Серверное подписание, XAdES для XML | ✅ Рабочий |
| `cryptopro-node-cades` | CSP 5.0, серверная работа | ✅ Рабочий |
| `node-cryptopro` | Генерация ключей, контейнеры | ✅ Рабочий |
| `crypto-pro-js` | Браузерный плагин, createXMLSignature | ✅ (клиент) |
| `xmldsigjs` | XML Digital Signature, TypeScript | ✅ Кроссплатформенный |

**Архитектура серверной подписи:**
```
TMS API → n-cryptopro → CryptoPro CSP 5.0 → USB-токен с ключом
                                              (или облачное хранилище)
```

### Вариант 2: КриптоАРМ

- Десктопное приложение для ручного подписания
- Не подходит для автоматизации в TMS
- **❌ Не рекомендуется** для нашего проекта

### Вариант 3: VipNet CSP

- Альтернативный криптопровайдер от ИнфоТеКС
- Менее распространён, чем CryptoPro
- Ограниченная поддержка в Node.js библиотеках
- **⚠️ Резервный вариант** (только если CryptoPro недоступен)

### Вариант 4: Облачная ЭП (Контур.Крипто)

**Контур.Крипто** предлагает RESTful API для подписания документов:

```
POST → Контур.Крипто API → подписанный документ
```

| Характеристика | Значение |
|---------------|----------|
| API | REST, SDK для Python/Java |
| Формат подписи | CMS (PKCS#7), XAdES (XML) |
| Хранение ключей | Облако (HSM-модуль оператора) |
| Без установки ПО | ✅ |
| Стоимость | Включена в тариф ЭДО оператора |

### Вариант 5: Облачная КЭП от ФНС (эксперимент)

С 25.12.2024 ФНС запустила эксперимент по облачной КЭП:
- Мобильное приложение «Подпись для бизнеса»
- Привязка к устройству (не переносится)
- Пока только для налоговой/бухгалтерской отчётности
- **⏳ Ожидаем** расширения на ЭПД (2026–2027)

## 🏆 Рекомендация

### Стратегия: **Контур.Крипто API + CryptoPro CSP как fallback**

1. **Основной путь**: Подписание через **API Контур.Крипто** (REST) → облачное хранение ключей на стороне оператора → нет необходимости устанавливать CryptoPro CSP на наш сервер

2. **Резервный путь**: Если потребуется серверное подписание без оператора → установить **CryptoPro CSP 5.0** + **n-cryptopro** для Node.js

3. **Для водителей**: **УНЭП через Госключ** (упрощённая подпись для подтверждения приёмки/доставки) — уже поддерживается ГИС ЭПД
