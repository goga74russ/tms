## 1. Вход в систему

**Страница:** `/login`

1. Откройте адрес системы в браузере
2. Введите **email** и **пароль**
3. Нажмите **«Войти»**

После входа вы увидите **боковое меню** с пунктами, доступными вашей роли. Каждый пользователь видит только свои разделы.

## 2. Роли и доступ

В системе **10 ролей**. Каждая роль видит свой набор страниц:

| Роль | Меню | Описание |
|------|------|----------|
| **Логист** | Заявки, Рейсы, Путевые листы, Автопарк, Контрагенты, Водители | Создаёт заявки, управляет грузами |
| **Диспетчер** | Диспетчерская, Заявки, Рейсы, Путевые листы, Автопарк, Водители | Назначает ТС и водителей, выпускает на линию |
| **Механик** | Техосмотр, Путевые листы, Автопарк, Ремонты, Водители | Проводит техосмотр ТС |
| **Медик** | Медосмотр, Путевые листы | Проводит предрейсовый медосмотр |
| **Водитель** | — (мобильное приложение) | Выполняет рейсы, отмечает точки |
| **Ремонтная служба** | Ремонты | Выполняет заявки на ремонт |
| **Бухгалтер** | Контрагенты, Финансы, Тарифы, Аналитика | Выставляет счета, контролирует оплаты |
| **Руководитель** | Автопарк, Финансы, KPI, Тарифы, Аналитика, Водители, Импорт | Смотрит KPI и принимает решения |
| **Клиент** | Портал клиента | Только просмотр статуса своих грузов |
| **Администратор** | Все разделы + Админ-панель | Полный доступ ко всей системе |

## 3. Логист — Заявки

**Страница:** `/logist`

### Что видит логист

- **Kanban-доска** — все заявки разбиты по статусам (колонки):
  - Черновик → **В работе** → Назначена → В пути → Доставлена

### Как создать заявку

1. Нажмите **«+ Новая заявка»**
2. Заполните форму:

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Контрагент | ✅ | Выбор из списка клиентов |
| Описание груза | ✅ | Что везём |
| Вес (кг) | ✅ | Масса груза |
| Объём (м³) | — | Объём в кубометрах |
| Кол-во мест | — | Сколько паллет / коробок |
| Ярусная загрузка | — | Можно ли грузить в 2-3 яруса |
| Температура (°C) | — | Для рефрижераторов: мин и макс |
| Тип загрузки | — | Задняя / боковая / верхняя |
| Гидроборт | — | Нужен ли гидроборт |
| Адрес погрузки | ✅ | Откуда забираем |
| Дата погрузки | — | Когда забираем |
| Адрес выгрузки | ✅ | Куда доставляем |
| Дата выгрузки | — | Когда доставляем |
| Примечания | — | Дополнительная информация |

3. Нажмите **«Создать заявку»** — она появится в колонке **Черновик**

### Как подтвердить заявку

- Перетащите карточку из **Черновик** → **В работе**
- Или нажмите кнопку **«Подтвердить»** на карточке

### Создание из шаблона

Для типовых заявок: кнопка **«Из шаблона»** — копирует существующую заявку с возможностью изменить поля.

## 4. Диспетчер — Рейсы и Назначения

### Диспетчерская (`/dispatcher`)

Рабочий экран диспетчера:
- Список подтверждённых заявок (статус «В работе»)
- Доступные ТС и водители
- Создание рейса: выбрать заявки + ТС + водителя

### Рейсы (`/trips`)

Список всех рейсов с фильтрацией по статусам:

| Статус | Означает |
|--------|----------|
| Планирование | Рейс создан, ТС/водитель назначены |
| Назначен | ТС и водитель подтверждены |
| Осмотр | На медицинском/техническом осмотре |
| Путевой выдан | Путевой лист оформлен, готов к выезду |
| Погрузка | Машина на точке погрузки |
| В пути | Машина в дороге |
| Завершён | Рейс выполнен, машина вернулась |
| Выставлен | Счёт клиенту выставлен |

### Как создать рейс

1. На странице **Диспетчерской** нажмите **«Создать рейс»**
2. Выберите:
   - Подтверждённые заявки (можно несколько — консолидация)
   - ТС из списка доступных
   - Водителя
   - Прицеп (если нужен)
3. Укажите плановый пробег и дату выезда
4. Нажмите **«Создать»**

## 5. Путевые листы

**Страница:** `/waybills`

Путевой лист — **главный документ рейса**. Содержит:
- Данные о ТС и водителе (водителях)
- Одометр при выезде и возврате
- Топливо при выезде и возврате
- Привязку к осмотрам (мед + тех)
- Расходы (план и факт)

| Статус | Означает |
|--------|----------|
| Оформлен | Путевой создан, ожидает осмотров |
| Открыт | Осмотры пройдены, машина на линии |
| Закрыт | Машина вернулась, данные заполнены |

### Расходы путевого листа

К путевому листу привязываются все затраты рейса:
- 🛢️ Топливо (план/факт литры и стоимость)
- 🛣️ Платон
- 🅿️ Стоянки
- ⚠️ Штрафы
- 🔧 Ремонт в пути
- 📦 Прочие расходы

Каждый расход имеет **плановую** и **фактическую** сумму для сравнения.

## 6. Механик — Техосмотр

**Страница:** `/mechanic`

### Проведение предрейсового техосмотра

1. Выберите ТС для осмотра
2. Заполните **чек-лист** (шины, тормоза, огни, документы, аптечка и т.д.)
3. Для каждого пункта: **ОК** или **Неисправность** (с фото и комментарием)
4. Вынесите решение:
   - ✅ **Допущен** — ТС готово к рейсу
   - ❌ **Не допущен** — автоматически создаётся заявка на ремонт

### Инциденты

Если осмотр **не пройден** — система создаёт **инцидент**:
- Тяжесть: низкая / средняя / критическая
- Критический инцидент **блокирует** выпуск на линию
- Инцидент виден диспетчеру и логисту

---

## 7. Медик — Медосмотр

**Страница:** `/medic`

### Проведение предрейсового медосмотра

1. Выберите водителя
2. Заполните данные:
   - 💉 Артериальное давление (верхнее / нижнее)
   - ❤️ Пульс
   - 🌡️ Температура
   - 🧪 Алкотест (отрицательный / положительный)
   - Жалобы, общее состояние
3. Вынесите решение:
   - ✅ **Допущен** — водитель готов к рейсу
   - ❌ **Не допущен** — создаётся инцидент

> 🔒 Медицинские данные **защищены** по 152-ФЗ. Диспетчер видит только факт допуска (да/нет), но не медданные. Все обращения к медданным логируются.

## 8. Автопарк

**Страница:** `/fleet`

### Карточка ТС

Каждое ТС содержит:

| Раздел | Поля |
|--------|------|
| **Основное** | Госномер, VIN, марка, модель, год, тип кузова |
| **Характеристики** | Грузоподъёмность (кг), объём (м³), гидроборт |
| **Документы** | ТО, ОСАГО, тахограф — сроки действия |
| **Расходники** | Топливная карта, транспондер, норма расхода |
| **Обслуживание** | Следующее ТО (дата/пробег), текущий одометр |

### Статусы ТС

| Статус | Цвет | Описание |
|--------|------|----------|
| Доступен | 🟢 | Готов к назначению |
| Назначен | 🔵 | Привязан к рейсу |
| В рейсе | 🟡 | На линии |
| На ТО | 🟠 | На плановом обслуживании |
| Неисправен | 🔴 | Требует ремонта |
| Заблокирован | ⚫ | Запрет эксплуатации |

### Прицепы

Прицепы — отдельная сущность со своим:
- Госномером, VIN, типом (тент, борт, реф, цистерна, площадка, контейнер)
- Сроками ТО, ОСАГО
- Привязкой к тягачу (может меняться)

### Штрафы и Пропуски

- **Штрафы ГИБДД** — регистрация, привязка к ТС/водителю, статус (новый → подтверждён → оплачен / обжалован)
- **Пропуска** — зоны (МКАД, ТТК, город), номер, срок действия

## 10. Ремонтная служба

**Страница:** `/repair`

### Заявки на ремонт

Заявки создаются:
- **Автоматически** — при непрохождении техосмотра
- **Вручную** — механиком или водителем

| Статус | Описание |
|--------|----------|
| Создана | Новая заявка |
| Ожидание запчастей | Нужны детали |
| В работе | Ремонт идёт |
| Выполнена | Ремонт завершён |

Каждая заявка содержит:
- Описание неисправности + приоритет (низкий / средний / высокий / критический)
- Использованные запчасти (название, количество, стоимость)
- Общая стоимость ремонта
- Фотоматериалы

## 12. Финансы — Счета и Акты

**Страница:** `/finance`

### Автоматический расчёт стоимости

При завершении рейса система рассчитывает стоимость по тарифу:
- За километр, за тонну, за час, фиксированный, комбинированный
- С коэффициентами: ночной, выходной, срочный
- Простой (бесплатный лимит + ставка за час)
- НДС (включён / не включён)

### Счета

| Статус | Описание |
|--------|----------|
| Черновик | Создан, не отправлен |
| Отправлен | Выставлен клиенту |
| Оплачен | Деньги получены |
| Просрочен | Срок оплаты истёк |

## 13. Тарифы

**Страница:** `/tariffs`

### Типы тарифов

| Тип | Формула |
|-----|---------|
| За км | `расстояние × ставка` |
| За тонну | `вес × ставка` |
| За час | `время × ставка` |
| Фиксированный | `фиксированная цена за маршрут` |
| Комбинированный | `фикс + ставка_за_км сверх порога` |

### Модификаторы

- Коэффициент ночной / выходной / срочный
- Бесплатный лимит простоя (мин) + ставка за час сверх
- Доплата за доп. точку
- Возврат (% от стоимости рейса)
- Штраф за отмену
- НДС (ставка + включён/нет)
- Минимальная стоимость рейса

## 14. Аналитика и KPI

### Аналитика (`/analytics`)

- **Предиктивное ТО** — алерты об истечении документов и сроков ТО
- **Дашборд расходов** — план vs факт

## 15. Портал клиента

**Страница:** `/client`

> ⚠️ Клиент **не создаёт** заявки. Всё создаёт **логист**.

### Что видит клиент

- **Список своих заявок** (фильтрация по `contractorId`)
- **Статус каждой заявки** — с цветовой индикацией:
  - Черновик → В работе → Назначена → В пути → Доставлена
- **Сводка** — активных заявок, доставлено за период

### Что клиент НЕ может

- Создавать заявки
- Редактировать заявки
- Видеть данные других клиентов
- Видеть внутреннюю информацию (водителей, ТС, финансы)

## 17. Мобильное приложение водителя

> Водитель работает **только через мобильное приложение** (React Native + Expo). Боковое меню на вебе для водителя скрыто.

### Основные функции

- 📍 Просмотр назначенного рейса и маршрута
- ✅ Отметка прибытия/убытия на точках (погрузка/выгрузка)
- 📸 Фотографирование (подпись, накладная, повреждения)
- 📡 Автоматическая отправка GPS-координат
- 🔄 **Офлайн-режим** — работа без интернета с синхронизацией при подключении

### Синхронизация данных

Приложение использует **WatermelonDB** для локального хранения:
- При наличии сети — автоматическая синхронизация
- При отсутствии — накопление записей → push при восстановлении
- Разрешение конфликтов по стратегии «last-write-wins»

---

## Приложение: Безопасность

| Механизм | Описание |
|----------|----------|
| **RBAC** | Ролевой доступ — каждый видит только своё |
| **RLS** | Row-Level Security — водитель видит только свои рейсы, клиент — свои заявки |
| **IDOR-защита** | Проверка владения ресурсом при каждом запросе |
| **152-ФЗ** | Медданные зашифрованы, доступ по ролям, аудит обращений |
| **JWT** | Токены авторизации с автообновлением |
| **Event Journal** | Все действия записываются в неизменяемый журнал событий |
