# Актуальный Sprint Backlog по итогам аудитов

**Дата:** 13.03.2026
**Основание:** аудит-файлы из `audits/` + сверка с текущим кодом после закрытия Sprint 9
**Назначение:** зафиксировать только реально незакрытые задачи и убрать уже устаревшие пункты старых аудитов

## Что исключено как уже закрыто

- Sprint 9 flow с путевыми листами `draft -> medical_check -> technical_check -> issued -> closed`
- Разделение осмотров на `pre_trip` и `periodic`
- `trip_orders` как рабочая many-to-many основа для консолидации
- `waybill_attachments`
- incidents-блокировки в выпуске и назначении
- batch cost вызов в `finance.service.ts` уже используется, поэтому пункт "N+1 не исправлен вообще" из части аудитов устарел

## Sprint 10: Бизнес-инварианты и P0-блокеры

**Цель:** закрыть разрывы между рейсом, путевым, заказом и инвойсом.

- [ ] Переписать `TarificationService` под multi-order рейсы: считать по `trip_orders`, убрать зависимость от `trip.order`, исправить проблемные поля и модификаторы.
- [ ] При `TripStatus.COMPLETED` переводить связанные `orders` в `delivered`.
- [ ] При `TripStatus.CANCELLED` освобождать ТС, водителя и связанные активы.
- [ ] Убрать обход выпуска через прямой переход в `WAYBILL_ISSUED`, если путевой не создан или не готов к выпуску.
- [ ] Жестко привязать предрейсовые мед- и техосмотры к конкретному `tripId`, чтобы допуск нельзя было использовать повторно в другом рейсе.
- [ ] Добавить в `trips` признак получения оригиналов документов и запретить биллинг без него.
- [ ] Понизить права роли `client` с `manage` до `read` для `Order`.
- [ ] Обновить `getTripById()` и связанные чтения рейса на `trip_orders` вместо legacy-чтения через `orders.tripId`.

**Основные модули:** `trips`, `waybills`, `finance`, `auth`, `orders`

## Sprint 11: Жесткие проверки и документный контур

**Цель:** сделать систему безопасной для ежедневной эксплуатации.

- [ ] Валидировать маршрут перед закрытием рейса и/или путевого: нельзя закрывать до завершения всех обязательных `routePoints`.
- [ ] Валидировать одометр при закрытии путевого: `odometerIn >= odometerOut`.
- [ ] Добавить `trailerId` в `trips` и `waybills`, встроить проверку доступности прицепа в назначение рейса.
- [ ] Ввести авто-блокировку ТС в `maintenance` при проваленном техосмотре.
- [ ] Блокировать назначение водителя при просроченной `medCertExpiry`.
- [ ] Перевести web-auth с `localStorage` на `httpOnly` cookie.
- [ ] Сделать реестр возврата первички: оригиналы ТТН/УПД, сканы, статус сдачи документов.
- [ ] Исправить ТТН/PDF/ЭТрН-реквизиты: корректный грузополучатель, отсутствующие обязательные поля, endpoint для акта.
- [ ] Ограничить доступ к медицинским деталям и проверить RLS/permissions для `waybillDrivers`, attachments и связанных sprint9 CRUD.
- [ ] Добавить отсутствующие FK и нормализовать проблемные связи, включая `repairRequests.inspectionId` и `invoices.tripIds`.

**Основные модули:** `documents`, `trips`, `waybills`, `fleet`, `auth`, `finance`

## Sprint 12: Финансовая связность и управленческий учет

**Цель:** связать операционный контур с удержаниями, штрафами и себестоимостью.

- [ ] Создать модуль `Claims` для претензий по порче, опозданию и другим спорным случаям.
- [ ] Связать `fines` и incidents с финансовым контуром: удержания, корректировки, спорные суммы.
- [ ] Добавить в тарификацию реальные расходы по Платону, транспондерам и связанным дорожным платежам.
- [ ] Довести `generateInvoices` и смежные выборки до нормальной batch-модели без деградации на объеме.
- [ ] Добавить составные индексы под реальные фильтры и массовые расчеты.
- [ ] Усилить append-only гарантию журнала событий.
- [ ] Реализовать модуль РТО/тахографа: остаток часов, hard/soft блокировки назначений, контроль переработок.

**Основные модули:** `finance`, `events`, `tachograph`, `claims`
